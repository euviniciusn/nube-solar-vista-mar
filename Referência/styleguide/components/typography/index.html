Typography

Custom component for consistent text styling across the design system. Built following shadcn/ui typography patterns.

import { Typography } from "@/components/ui/typography"

Heading variants from h1 to h4.

variant="h2"

variant="h3"

variant="h4"

Blockquote

Styled blockquote for quotations.

"After all," he said, "everyone enjoys a good joke, so it's only fair that they should pay for the privilege."
<Typography variant="blockquote">
  "After all," he said, "everyone enjoys a good joke."
</Typography>

Inline Code

Monospace inline code snippets.

Use the Typography component with variant="inline-code" for inline code.

<Typography variant="inline-code">code here</Typography>

Custom Element (as prop)

Override the rendered HTML element with the 'as' prop.

H3 styles rendered as a span

Useful when you need visual styling of one element but semantic meaning of another.

// Renders as <span> with h3 styles
<Typography variant="h3" as="span">
  H3 styled span
</Typography>

// Renders as <label> with small styles
<Typography variant="small" as="label">
  Field Label
</Typography>

Full Article Example

All typography variants combined in an article layout.

Taxing Laughter

Once upon a time, in a far-off land, there was a very lazy king who spent all day lounging on his throne.

The King's Plan

The king thought long and hard, and finally came up with a brilliant plan: he would tax the jokes in the kingdom.

"After all, everyone enjoys a good joke, so it's only fair that they should pay for the privilege."

The Joke Tax

The king's subjects were not amused. They grumbled and complained, but the king was firm:

  • 1st level of puns: 5 gold coins
  • 2nd level of jokes: 10 gold coins
  • 3rd level of one-liners: 20 gold coins

The moral of the story is: never underestimate the power of a good laugh.

Accessibility

  • Each variant maps to its semantic HTML element by default
  • Use as prop when visual style differs from semantic meaning
  • Maintain proper heading hierarchy (h1 → h2 → h3 → h4)
  • Don't skip heading levels for visual styling - use as instead
  • All text meets WCAG 2.1 AA contrast requirements against the design system backgrounds
